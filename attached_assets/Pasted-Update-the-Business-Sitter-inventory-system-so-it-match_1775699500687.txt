Update the Business Sitter inventory system so it matches the attached inventory sheet exactly.

Goal:
The current inventory model is wrong. Replace the current Business Sitter shift inventory structure with the spreadsheet-based structure from the attached inventory workbook.

Source layout to mirror:
- Column A = category headers and item labels
- Column C = Starting Inventory value
- Column D = unit type ("G" or "#")
- Column E = Ending Inventory value
- The sheet groups items by category blocks with blank spacer rows between sections
- Example sections in the attached file include:
  - Premium Smoke Collection
  - Intimate Gel Collection
  - AquaSilk Water-Based Lubricants
  - Crimson Brick Condoms
  - Obsidian Edge Collection
  - Accessories
  - Pharmacy
  - Petty Cash
- Starting inventory values are currently stored in the spreadsheet under Column C and ending inventory under Column E. :contentReference[oaicite:0]{index=0}

Business requirement:
When the Business Sitter clocks in, the “starting inventory” must come from this inventory structure.
When the Business Sitter clocks out, the system must calculate and save the “ending inventory” against the same structure.
Do not use the current generic inventory row model if it does not match the spreadsheet layout.

Implement these changes:

1. Create a canonical inventory template model
Build a server-side inventory template that matches the attached sheet structure exactly, including:
- section/category name
- item label
- unit type
- starting quantity default
- ending quantity
- display order
- whether the row is a section header, item row, spacer row, or special row like petty cash

Suggested internal shape:
- id
- section_name
- item_name
- row_type: "section" | "item" | "spacer" | "cash"
- unit_type: "G" | "#"
- starting_quantity_default
- ending_quantity_default
- display_order
- is_active

2. Seed the inventory template from the attached spreadsheet
Create a seed/migration so the Business Sitter inventory template reflects the attached workbook rows.
Mirror the row order from the spreadsheet.
Preserve section headers and blank separator rows in UI rendering.

3. Replace the current Business Sitter clock-in flow
On clock-in:
- load the inventory template
- prefill the starting inventory form with the template’s starting quantities from the attached workbook
- allow edits before clock-in is finalized
- save the actual starting inventory snapshot for that shift

Do not make the sitter manually build inventory rows from scratch.
The current “Add Item / Item name / Unit price / Start Qty” freeform flow should be replaced or made optional only for admins if needed.

4. Replace the current active shift inventory panel
During an active shift, show inventory in the same grouped layout as the spreadsheet:
- section headers
- item rows
- unit type shown
- starting qty
- sold qty
- ending qty / remaining qty
Keep the UI visually grouped by section.

5. Replace clock-out logic
On clock-out:
- calculate quantity sold per item from actual fulfilled orders during the shift
- subtract sold quantity from starting quantity to derive ending quantity
- store the ending inventory snapshot
- show the shift summary in the same grouped structure
- support both gram-based and count-based items

Formula:
ending_quantity = starting_quantity - quantity_sold

Do not allow negative ending inventory without flagging it.

6. Support item-to-product mapping
Add or fix mapping between Business Sitter inventory rows and product/menu items so sold quantities reduce the correct row.
Some inventory rows may map to one product, while some may be informational headers and should not map to any product.

Suggested fields:
- catalog_item_id nullable
- alavont_id nullable
- lucifer_cruz_name nullable
- inventory_item_name
- deduction_unit_type
- deduction_quantity_per_sale

7. Handle special cases from the attached file
Support:
- gram-based rows like Powder or Total In Grams
- count-based rows like bottles, pills, cartridges, accessories
- petty cash as a tracked special row
- section rows that are not inventory inputs
- rows that currently say “Total pills” or similar labels
Normalize labels where useful internally, but keep display labels matching the spreadsheet.

8. Admin inventory management
Create an admin-only “Inventory Template” screen where admins can:
- edit default starting quantities
- edit labels
- edit unit types
- reorder rows
- toggle rows active/inactive
- re-seed from the attached inventory structure if needed

9. Data model changes
Add tables or update schema for:
- inventory_templates
- shift_inventory_snapshots
- shift_inventory_snapshot_items

Each shift should store an immutable snapshot of:
- starting inventory
- ending inventory
- qty sold
- unit type
- section/item labels at time of shift

10. Validation rules
- section rows cannot have sold quantity
- item rows must have unit type if tracked
- starting quantity must be numeric
- ending quantity must be numeric
- grams can be decimal
- count rows should default to integers unless explicitly configured otherwise
- negative results should be flagged for review

11. UI requirements
Business Sitter screen should show:
- section title rows styled clearly
- item rows underneath
- starting qty
- sold qty
- ending qty
- unit column
- shift summary grouped exactly like the source inventory sheet

12. Migration / compatibility
Migrate away from the current generic shift inventory row system carefully.
Do not break existing shift history.
For old shifts, preserve existing data.
For new shifts, use the new template-based structure.

13. Testing
Write tests for:
- loading inventory template on clock-in
- saving starting inventory snapshot
- deducting sold quantities correctly from orders
- computing ending inventory
- handling gram rows and count rows
- preserving section order
- negative inventory flagging
- petty cash row preservation

Important:
Use the attached inventory workbook as the source of truth for the row structure and starting inventory defaults.

Add the Alavont logo to all printable receipt outputs in the MyOrder.fun system.

Goal:
Use the provided ASCII logo art as the header for:
1. customer receipts
2. starting inventory receipts
3. clock-out / ending inventory sheets

Important:
- Use the provided ASCII logo text file as the source
- Keep it editable so I can replace/update the logo later without changing business logic
- Make it safe for thermal printing
- Do not hardcode the full logo string directly inside receipt builders

Implementation requirements:

1. Create a printable logo asset loader
Add a reusable helper that loads the ASCII logo from a text file, for example:
- /artifacts/print/logo.txt
or
- /server/print/templates/logo.txt

The helper should:
- read the file from disk
- return a fallback string like "ALAVONT THERAPEUTICS" if the file is missing
- normalize line endings
- optionally trim trailing whitespace

Suggested helper behavior:
- getPrintableLogo()
- fitPrintableLogo(maxWidth)

2. Add a width-fitting helper
Thermal printers have limited width, so add a helper that:
- trims or safely fits lines to the printer width
- defaults to receipt width appropriate for the configured printer
- avoids wrapping the logo badly

Suggested default widths:
- 32 chars for 58mm
- 42 or 48 chars for 80mm

The helper should:
- split by line
- trim trailing whitespace
- optionally truncate lines to max width
- rejoin with newline

3. Update customer receipt template
Update the customer receipt builder so the logo prints at the top before:
- receipt title
- order metadata
- line items
- totals

Structure should be:
[logo]
================================
SECURE ORDER RECEIPT
================================
Order details...
Items...
Totals...

Keep receipt output text-based for bridge printing.

4. Update starting inventory receipt template
Update the starting inventory / shift start printable template so the logo appears at the top.

Structure should be:
[logo]
================================
STARTING INVENTORY
================================
Operator
Date
Inventory sections/items

5. Update ending inventory / clock-out template
Update the ending inventory / shift summary printable template so the logo appears at the top.

Structure should be:
[logo]
================================
ENDING INVENTORY
================================
Operator
Date
Start / End / Sold values

6. Keep logo editable
Do not embed the ASCII logo directly in code.
Store it in a dedicated file so I can replace it later.

7. Add printer-width awareness
If the print renderer already knows paper width from printer config, pass that width into the logo-fitting helper so the logo can be constrained correctly.

8. Add a print preview route or dev utility
Add a simple way for admin/dev to preview the generated receipt text in-browser or in logs before sending to the bridge.
At minimum:
- test preview for customer receipt
- test preview for starting inventory
- test preview for ending inventory

9. Keep receipts discreet and clean
Do not print unnecessary sensitive internal fields.
Only include what belongs on that specific receipt type.

10. Files/components to implement
Suggested files:
- server/print/logo.ts
- server/print/templates/logo.txt
- server/print/templates/customerReceipt.ts
- server/print/templates/inventoryStart.ts
- server/print/templates/inventoryEnd.ts

11. Testing
Add tests for:
- missing logo file falls back safely
- logo file loads correctly
- logo width fitting works
- each receipt type includes logo at top
- output remains valid text for bridge printing

12. Do not break
- existing print bridge integration
- receipt printer config
- label printer config
- order print routing
- business sitter inventory print flow

Expected behavior:
Whenever a receipt or inventory sheet is rendered for printing, the Alavont ASCII logo appears at the top automatically.
